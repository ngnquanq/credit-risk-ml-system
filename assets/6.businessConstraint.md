## 🚫 Immediate Rejection Rules

### 1. Age Requirements
- **Field:** `DAYS_BIRTH`
- **Rule:** Age < 18 or Age > 75 years
- **Rationale:** Regulatory compliance - cannot lend to minors or high-risk elderly customers

### 2. Income Requirements
- **Field:** `AMT_INCOME_TOTAL`
- **Rule:** Below minimum threshold (e.g., < $15,000/year)
- **Rationale:** Insufficient income to support loan repayment

- **Field:** `AMT_CREDIT/AMT_INCOME_TOTAL` 
- **Rule:** Debt-to-income ratio > 40-50%
- **Rationale:** Excessive debt burden relative to income (expands on your 2*Income rule)

- **Field:** `AMT_ANNUITY/AMT_INCOME_TOTAL`
- **Rule:** Monthly payment > 30% of monthly income
- **Rationale:** Loan payment would create financial hardship

### 3. Employment Status
- **Field:** `NAME_INCOME_TYPE`
- **Rule:** "Unemployed" or "Student" (without co-signer)
- **Rationale:** Unstable income source

- **Field:** `DAYS_EMPLOYED`
- **Rule:** Unemployed (positive values indicate unemployment)
- **Rationale:** No current employment verification

### 4. Contact Information Requirements
- **Fields:** `FLAG_MOBIL` AND `FLAG_PHONE`
- **Rule:** Both = 0 (no mobile AND no phone)
- **Rationale:** No way to contact customer for collections/communication (relates to your "Customer must provide" requirement)

### 5. External Credit Scores
- **Fields:** `EXT_SOURCE_1`, `EXT_SOURCE_2`, `EXT_SOURCE_3`
- **Rule:** All scores below critical threshold (e.g., < 0.3)
- **Rationale:** Bureau data indicates severe credit issues

### 6. High-Risk Asset Indicators
- **Fields:** `CNT_CHILDREN` vs `AMT_INCOME_TOTAL`
- **Rule:** Too many dependents relative to income capacity
- **Rationale:** Limited disposable income after family expenses

- **Fields:** `FLAG_OWN_CAR` AND `FLAG_OWN_REALTY`
- **Rule:** Both = 0 (no car AND no property)
- **Rationale:** No assets available for collateral or stability indication

## 🏗️ Architecture Integration

### Pre-Screening Service Component
A **Business Rules Engine** should be implemented before ML Scoring:

```
API Service → Business Rules Engine → {
  ✅ Pass → ML Scoring Service
  ❌ Fail → Immediate Rejection Response
}
```

### Implementation Benefits
- **Performance:** Faster response times - no ML inference needed for obvious rejections
- **Cost Optimization:** Reduce ML API calls by approximately 20-30%
- **Compliance:** Enforce regulatory requirements consistently across all applications
- **Resource Management:** Reserve computational resources for viable loan candidates

## Configuration Management
- Rules should be externally configurable (environment variables, config files)
- Thresholds should be adjustable based on business policy changes
- All rejections should be logged with specific rule violations for audit trails

## Monitoring & Analytics
- Track rejection rates by rule type
- Monitor false negative rates (good customers rejected by rules)
- A/B test rule threshold adjustments
- Generate business intelligence reports on rejection patterns
